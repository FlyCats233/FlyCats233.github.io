我为什么要写博客（含搭建问题解决思路）

2026/7/3

博客，就是Web Log，后来简读成Blog，用于在网络上记录自己的任何东西，这就是博客的来源。在社交媒体远不发达的上个时代，人们通过博客在互联网上留下自己的痕迹。那时，互联网有门槛，博客更有门槛，但拥有这项技能的人，却更热衷于免费撰写一篇又一篇的博客，这些博客成为了从上个互联网时代流传下来的知识瑰宝。如今已鲜少有人会制作一个专属于自己的博客，更不会在一个没有流量的不知名网站上记录自己，更多人愿意在QQ微信、抖音B站等社交媒体上记录，但抛开这些流量话题，对自己而言，对我而言，博客在如今，不仅是一种年少轻狂、标新立异的自我表达，同时也是一种不会担心被凝视的表达。现在的互联网社区早已乌烟瘴气，而在陌生而又古老的博客面前，人们也许会保留一份自己心中的平静。

平静，独自一人的舞台

其实一切的契机是在实习上班期间，有一些没事摸鱼的空闲，但这时候不能光明正大在工位玩游戏刷视频，所以我就打算用电脑学点好玩的，比如重新拿起前年秋天程序设计课上的cpp大作业，就是一个比较粗糙的Tetris俄罗斯方块小游戏，我打算给它进行一次重构，做成一个玩法更新颖、有独立UI的游戏，当然这个过程少不了ai辅助，毕竟能偷懒谁不干呢(*￣3￣)╭其实我还想靠ai设计做一个Minecraft的模组，不过那都是后话了。这时我在想，怎么记录我的这些好主意和制作经验呢？GitHub可以用来托管代码和方便更新迭代，那么一个程序员经常会有的另一个东西就是自己的博客网站了，博客网站又可以免费挂载在GayHub page上，这样服务器和网址备案的问题又解决了，简单查阅了一下教程，用Hexo实现静态个人博客很方便，主题自由选取，何乐而不为呢？于是说干就干，中间也遇到了很多困难，不过在搜索引擎和ds老师的指导下全部解决~

具体的搭建每一步我就不细说了，我也是按照网上的教程一步一步来的，这里介绍一些我遇到过的问题解决思路

  1. 根目录配置文件_config.yml
    # Deployment
## Docs: https://hexo.io/docs/one-command-deployment
deploy:
type: git
repository: git@github.com:FlyCats233/FlyCats233.github.io.git
branch: main # <--你的分支名
    因为现在版本的GitHub主分支默认名称从Master变成了main，所以需要修改配置文件中的branch值为main，如果不想产生额外分支的话。
  2. 主题theme
    theme/my_theme/_config.yml是主题的配置文件，但是Hexo允许在根目录下新建一个_config.my_theme.yml文件，用来便携地修改这个主题的配置，默认将theme/my_theme/_config.yml的内容与_config.my_theme.yml的内容进行合并，如果冲突，后者的优先级更高。所以，我将主题的原配置文件的所有实际内容删除，只留下空占位，然后在根目录下新建一个_config.my_theme.yml进行配置，这样就不需要每次都进入特定主题文件夹下配置了。
  3. 主题配置文件_config.my_theme.yml，这里我用的Particle主题
    # Gitalk
# https://github.com/gitalk/gitalk
gitalk:
enable: true
clientID: xxx # GitHub Application Client ID
clientSecret: xxx # GitHub Application Client Secret
repo: gitalk-comments # The name of repository of store comments
owner: FlyCats233 # GitHub repo owner
admin: FlyCats233 # GitHub repo owner and collaborators, only these guys can initialize github issues
language: zh-CN # en, zh-CN, zh-TW, es-ES, fr, ru, de, pl and ko are currently available.
    Gitalk是Hexo的一个评论插件，部署起来比较简单，主要是language会有一个问题，如果把默认的en改为zh-CN并上传，会导致GitHub上的Action发生错误，从而部署page失败
